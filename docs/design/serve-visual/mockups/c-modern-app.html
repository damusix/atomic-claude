Caching Patterns

Cache-aside vs write-through tradeoffs — when to read-through on miss, and where the error contract lives when a write fails.

Cache-aside

Read-through on miss, write to cache after DB. Used by Alpha Service for rate tables.

Write-through

Write to cache and DB synchronously. See API Conventions for the error contract when the cache write fails.

Latency — cache-aside adds a round-trip only on miss; subsequent reads stay hot.

Consistency — write-through pays on every write to keep cache and DB in lockstep.

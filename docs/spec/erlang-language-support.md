# Spec: Erlang tree-sitter language support

## Goal

Add Erlang (`.erl`, `.hrl`) as a supported tree-sitter language so Erlang source extracts module/function/record symbols (currently unsupported → 0 nodes). Independent of Elixir (both BEAM, but distinct grammars). v1 is language symbols only — OTP-behaviour "framework" resolution (gen_server/supervisor trees) is explicitly out of scope.

## Approach

Tree-sitter, not regex. Grammar spike (2026-06-18) confirmed `WhatsApp/tree-sitter-erlang` tag `0.19` (commit `836aa2b6c3af2c7cef3f84049b0ed6d44485a870`) ships `parser.c` at `LANGUAGE_VERSION 14` — inside the binding ABI ceiling — plus a C `scanner.c` (no C++ scanner), and exports `tree_sitter_erlang`. Same shape as the working dart/luau/objc/pascal external pins; no ABI wall. Regex fallback not used.

Follow `atomic/internal/codeintel/tsbinding/CLAUDE.md` "Add a new grammar" (into the wasm, then into the engine). The just-merged Elixir support is the closest reference for the engine-wiring shape, but the EXTRACTOR differs: Erlang's grammar has dedicated node types (module attributes, function clauses, records), not Elixir's uniform `call` nodes — probe the real node types before writing the config.

## Success criteria

- `lib/ts.wasm` rebuilds with the Erlang grammar compiled in and exports `tree_sitter_erlang`.
- A real `.erl` file parses to a non-empty named-node tree.
- `.erl` and `.hrl` route to Erlang in the indexer.
- Extraction of a representative Erlang fixture yields, at minimum: `-module(name).` → module symbol; function clauses (`fname(...) -> ...`) → function symbols with name/arity identity (Erlang identity is name+arity, e.g. `foo/2`); `-record(name, {...}).` → struct symbol (fields → field nodes where cheap). Edges where cheap & unambiguous: `-export([f/1,...]).` → mark functions exported (IsExported); `-import` → import edges; `-behaviour(mod).` → reference/implements edge; remote (`mod:fun(...)`) and local (`fun(...)`) calls → call edges; `-define(MACRO,...)` → constant.
- `go test -count=1 ./internal/codeintel/extraction/... ./internal/codeintel/types/... ./internal/codeintel/indexer/...` green, including a new Erlang extraction test asserting the symbols above (fails if Erlang extraction regresses to 0).
- Render/bundle parity unaffected; `src/` never committed; `grammars.json` + `Makefile` + `lib/ts.wasm` + Go changes commit together. `types.AllLanguages` + the `TestLanguageCount` canonical list (appendix C in `docs/spec/code-intel-engine.md`) updated for the new language.

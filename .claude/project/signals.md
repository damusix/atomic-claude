# Project signals

## Framework & runtime

- **Go 1.25**, single module at [`atomic/go.mod`](../../atomic/go.mod) (`module github.com/damusix/atomic-claude/atomic`). CLI binary named [`atomic`](../../atomic).
- **codeintel tsbinding sub-module**: [`atomic/internal/codeintel/tsbinding/go.mod`](../../atomic/internal/codeintel/tsbinding/go.mod) (`module github.com/malivvan/tree-sitter`, go 1.23.4) â€” wazero + tree-sitter WASM binding, separate module.
- **External Go deps**: `gopkg.in/yaml.v3` (YAML frontmatter), `github.com/tailscale/hujson` (lenient JSON for hooks/settings), `github.com/pelletier/go-toml/v2` (TOML config), `github.com/charmbracelet/huh` (interactive TUI prompts), `github.com/tetratelabs/wazero` (WASM runtime for tree-sitter), `github.com/modelcontextprotocol/go-sdk` (MCP server).
- **Go embed**: `//go:embed all:bundle` in [`atomic/internal/embedded/bundle.go`](../../atomic/internal/embedded/bundle.go). `all:` prefix required to include [`commands/_templates/`](../../commands/_templates) (underscore prefix skipped by plain `bundle` glob).
- **goreleaser**: multi-platform release â€” `linux/darwin x amd64/arm64`. CGO disabled. Version from `internal/version` via ldflags.
- **release-please**: automated changelog + tag on main push.
- **VitePress docs site**: [`package.json`](../../package.json) (`name=atomic-claude-docs`), [`package-lock.json`](../../package-lock.json), [`.vitepress/config.mts`](../../.vitepress/config.mts). Scripts: `docs:build`, `docs:dev`, `docs:preview`. Deployed via [`.github/workflows/docs.yml`](../../.github/workflows/docs.yml). Not part of the Go build.
- No web framework. SQLite used only by the code-intel engine at `<projectRoot>/.claude/.atomic-index/atomic.db`.

## Build / test / lint

All commands run from [`atomic/`](../../atomic) (CI: `working-directory: ./atomic`):

| Purpose | Command | Source |
|---------|---------|--------|
| Render command templates | `make render` (root) or `make -C atomic render` | [`atomic/Makefile`](../../atomic/Makefile), CI "Verify render is committed" |
| Regenerate embedded bundle | `go generate ./...` â†’ `make -C atomic bundle` | [`atomic/Makefile`](../../atomic/Makefile) target `bundle`, CI "Verify bundle is committed" |
| Run tests | `go test ./...` | [`atomic/Makefile`](../../atomic/Makefile) target `test` |
| Vet | `go vet ./...` | [`atomic/Makefile`](../../atomic/Makefile) target `vet` |
| Format check | `gofmt -l .` | [`atomic/Makefile`](../../atomic/Makefile) target `fmt` |
| Build binary | `go build -o ../bin/atomic ./cmd/atomic` | [`atomic/Makefile`](../../atomic/Makefile) target `build` |
| Tidy deps | `go mod tidy` | [`atomic/Makefile`](../../atomic/Makefile) target `tidy` |
| Release | `goreleaser release --clean` | [`.github/workflows/release.yml`](../../.github/workflows/release.yml) (on `v*` tag) |

CI gates: (1) `make render && git diff --exit-code` â€” stale [`commands/`](../../commands) fails. (2) `make bundle && git diff --exit-code` â€” stale `manifest.go` fails. Pipeline order: render must run before bundle.

The inferrer reads it when needed; sessions do not.

Prior contract (requiring both `deterministic-signals.md` and `signals.md`) is superseded per spec change 2026-05-26.

**Doctor hooks scope bug fixed**: `checks_hooks.go` `checkHooks` now passes `$HOME` as scopeRoot to `RunCheckHooksWith` â€” not `~/.claude`. Passing `~/.claude` caused `hooks.IsInstalled` to look for `~/.claude/.claude/settings.json` (double [`.claude`](..) segment). `RunCheckHooksWith(scopeRoot)` is exported for tests; `checks_hooks_internal_test.go` added for package-internal test coverage.

**Hooks inline-command migration**: [`atomic/internal/hooks/hooks.go`](../../atomic/internal/hooks/hooks.go) `Install` now registers the literal string `atomic hooks session-start` directly in `settings.json`'s `SessionStart` hook array â€” no wrapper script written. Legacy installs that registered `~/.claude/hooks/session-start-reminders.sh` are detected via `IsInstalled` `drifted=true` return and migrated away by `migrateLegacy` on the next `Install` call. `legacyScriptName` / `legacyHooksSubdir` constants retained solely for this migration path.

**`go:embed all:bundle` requirement (bundle)**: [`commands/_templates/`](../../commands/_templates) starts with `_` â€” excluded by the default embed glob. `all:bundle` overrides this. Any new underscore-prefixed directory under `embedded/bundle/` needs this same consideration.

**Pipeline order is load-bearing (bundle)**: `make render` must precede `make bundle`. The pre-commit hook stages 1 and 2 enforce this order; stage 1 re-stages both [`commands/`](../../commands) and [`agents/`](../../agents) after render. CI runs the same two drift gates in order. Running only `make bundle` after editing a template embeds stale command or agent outputs.

**Agent template partial pipeline**: [`templates/agents/`](../../templates/agents) now mirrors [`templates/commands/`](../../templates/commands) â€” agent source files are rendered, not directly edited. Seven agent-specific partials live in [`templates/shared/`](../../templates/shared): `agent-tdd-signals`, `agent-signals-output`, `agent-shared-rules`, `agent-search-tooling` (now carries one-line bridge naming code-intel as top search tier), `agent-implementer-workflow`, `agent-atomic-voice` (response-voice rule â€” injected into all 9 agent templates), `agent-code-intel` (`atomic code` verb guidance: lead with `explore` for orientation, then targeted verbs; graceful degradation). `atomic-builder.md` and `atomic-surgeon.md` receive `agent-code-intel` via `agent-implementer-workflow`. `atomic-investigator.md`, `atomic-reviewer.md`, `atomic-haiku.md`, `atomic-signals-inferrer.md`, and `atomic-strategist.md` include `agent-code-intel` directly. `make render` produces `agents/*.md`; `make bundle` embeds them. Editing `agents/*.md` directly is overwritten on next render â€” edit `templates/agents/*.md` instead.

**Domain partitioning basis**: domains are vertical slices by feature concern, not horizontal layers by file type. Each domain file answers: "if I'm working on X, what artifacts, what CLI code, and what docs do I need?"

**Artifact additions checklist**: adding any new command/agent/skill requires updating the artifact file, [`CLAUDE.md`](../../CLAUDE.md), [`README.md`](../../README.md), relevant [`docs/reference/`](../../docs/reference) tables, `docs/spec/<topic>.md` if non-trivial, cross-references in other artifacts, running `make render` and `make bundle`, and `/refresh-signals`. See [`claude.local.md`](../../claude.local.md) for the full checklist with per-row guidance.

**`/refresh-signals` is the single idempotent entry point**: `/initialize-signals` was removed (commit 4011b30). `/refresh-signals` handles both first-run init and subsequent refreshes. The `atomic-signals` skill was removed and absorbed into `atomic-signals-inferrer` â€” the agent now owns the full pipeline (scan + infer + wire).

**`atomic validate artifacts` (rule A1, doctor domain)**: [`atomic/internal/cliusage/`](../../atomic/internal/cliusage) is the single source of truth for the [`atomic`](../../atomic) command surface â€” verb-path tokens, accepted `--flags`, descriptions. `main.go` renders `--help` from it; `validate/artifacts.go` uses it to lint artifact citations for unknown verbs/flags. Adding or removing a verb or flag requires updating `cliusage.go`; failing to do so causes `atomic validate artifacts` to either false-positive (flag a valid citation) or false-negative (miss an invalid one). The A1 rule fires as part of whole-repo `atomic validate` (no subcommand) and as `atomic validate artifacts [paths...]`.

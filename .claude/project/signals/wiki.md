# wiki

## What it does

Maintains a cross-repository knowledge base (the "wiki") for one realm of repos. Deterministic CLI verbs (`atomic wiki scan`, `atomic wiki stale`, `atomic wiki linkify`, `atomic wiki bucket`) scaffold, classify, fingerprint, and report freshness; `/refresh-wiki` drives the LLM pass — summarizing no-signals repos via `atomic-signals-inferrer` in wiki-output mode, synthesizing cross-cutting concerns incrementally, and running a bucket-synthesis pass for each bucket with a non-empty diff (dispatching `atomic-signals-inferrer` in bucket-synthesis mode, writing topic-keyed pages under `wiki/knowledge/`). A cheap session-start nudge and ship-time `.dirty` marker prevent unnoticed drift.

## Artifacts

- [`commands/refresh-wiki.md`](../../../commands/refresh-wiki.md) — `/refresh-wiki [root]` command: scan → stale → incremental LLM pass → bucket-synthesis phase → linkify → disposition report → `.dirty` clear → commit offer. Deterministic and judgment halves are fenced separately. On first run in a realm with no `<wiki-buckets>` block, prompts once for bucket names; blank response records `declined="true"` so the offer never re-fires.
- [`agents/atomic-signals-inferrer.md`](../../../agents/atomic-signals-inferrer.md) — rendered agent; wiki-output mode branch dispatched by `/refresh-wiki` to summarize no-signals repos (`target_repo` + `wiki_dir` required, fail loud on partial args); bucket-synthesis mode branch dispatched per bucket with a non-empty diff, writes topic-keyed pages under `wiki/knowledge/`.
- [`templates/commands/refresh-wiki.md`](../../../templates/commands/refresh-wiki.md) — source of truth for the refresh-wiki command; `make render` → [`commands/refresh-wiki.md`](../../../commands/refresh-wiki.md).
- [`templates/agents/atomic-signals-inferrer.md`](../../../templates/agents/atomic-signals-inferrer.md) — source of truth for the inferrer agent, including the wiki-output and bucket-synthesis branches.
- [`templates/shared/signals-gate.md`](../../../templates/shared/signals-gate.md) — partial composed by all ship verbs; invokes `atomic wiki mark-dirty` after signals refresh.

## CLI code

- [`atomic/internal/wiki/wiki.go`](../../../atomic/internal/wiki/wiki.go) — core package: `Scan(root, Options)` entry point. Stages: collision check → parse prior entries → `discoverMembers` (recursive walk from root's children, stops at each `.git` boundary, skips `node_modules`/`dist`/`build`/`target`/`vendor`/`.worktrees`/`tmp`/`.git` and the wiki dir itself, root never a member) → `classifyMembers` → scaffold → `writeWikiScanBlock` → `writeMembersSection`. `Options.Clock` is injectable for deterministic test output.
- [`atomic/internal/wiki/wiki.go`](../../../atomic/internal/wiki/wiki.go) (`classifyMembers`) — four-rule precedence per member: (1) prior `summarized` + summary file exists on disk → keep `summarized`; (2) [`.claude/project/signals.md`](../signals.md) exists → `indexed` (signals win over a leftover summary); (3) `discoverSummary` finds `wiki/repos/<name>.md` or `wiki/repos/<name>/` with at least one `.md` → `summarized`; (4) otherwise → `pending`. Rule 3 (disk-discovery) was added 2026-06-11 to make `summarized` reachable on a closing re-scan after `/refresh-wiki` writes summaries.
- [`atomic/internal/wiki/wiki.go`](../../../atomic/internal/wiki/wiki.go) (`writeMembersSection`) — writes a managed `## Members` linked section into `wiki/index.md` between `<!-- wiki-members:start -->` / `<!-- wiki-members:end -->` markers, spliced idempotently. Link targets per status: `indexed` → `../<repo>/.claude/project/signals.md`; `summarized` → the recorded `SummaryPath` (`repos/<repo>.md` or `repos/<repo>/`); `pending` → `../<repo>/`.
- [`atomic/internal/wiki/bucket.go`](../../../atomic/internal/wiki/bucket.go) — CP1 bucket manifest core. `WalkBucket(dir)` returns sorted `<relpath>\t<sha256hex>` entries; excludes `index.md` at bucket root, `.DS_Store`/`Thumbs.db`, and `skipDirs` subtrees. `RegisterBucket(wikiDir, name)` creates `wiki/.buckets/<name>/`; refuses name=`wiki` or already-registered. `BucketDiff(wikiDir, name, dir)` walks dir, writes `wiki/.buckets/<name>/current`, diffs against baseline; all-added when no baseline exists (first call). `bucketDiffReadOnly` is the same diff without writing `current` — used by `list` (status verb, no side effects). `PromoteBucket(wikiDir, name, dir)` recomputes live walk; first promote: walk→baseline; subsequent: baseline→previous, walk→baseline.
- [`atomic/internal/wiki/bucket_registry.go`](../../../atomic/internal/wiki/bucket_registry.go) — CP2 registry primitives. `spliceBucketEntry` maintains the `<wiki-buckets>` block in `wiki/index.md` — idempotent splice of `<bucket name="…" path="…"/>` entries; removes `declined="true"` attribute when a new bucket is added; creates the file if absent. `readBucketEntries` parses the block and returns `(name, path)` pairs. `writeCaptureSurfacesSection` appends a `## Capture surfaces` section to the realm [`CLAUDE.md`](../../../CLAUDE.md) (written once on first `bucket add`; heading not duplicated; one bullet per bucket with `<!-- describe what this bucket is for -->` placeholder). `createBucketIndexStub` writes `<bucketDir>/index.md` if absent (heading + `## Conventions` placeholder); no-op if file exists.
- [`atomic/internal/wiki/action.go`](../../../atomic/internal/wiki/action.go) — `WikiAction` / `wikiAction` — exported/unexported testable seam for `atomic wiki` subcommand dispatch. Switch on verb: `scan`, `stale`, `stamp`, `mark-dirty`, `linkify`, `bucket`. `wikiBucketAction` sub-dispatches `add`, `list`, `diff`, `promote`. `wikiStampAction` dispatches three modes: summary (`--repo`), concern (`--root` + `--cites`), knowledge (`--knowledge` + `--sources`). Knowledge topic names must match `[a-z0-9][a-z0-9-]*\.md`; non-conforming names are skipped with exit 0.
- [`atomic/internal/wiki/stale.go`](../../../atomic/internal/wiki/stale.go) — `Stale(root, out)` entry point for `atomic wiki stale`. Three check sections: (1) membership/status drift against `<wiki-scan>` block; (2) per-artifact content drift (summary `reflects_rev` vs git HEAD; concern `reflects:` entries vs recomputed fingerprints); (3) bucket staleness — reads `<wiki-buckets>` block, calls `bucketDiffReadOnly` per entry, emits `STALE bucket <name>` when diff non-empty. Walk/I/O errors on buckets escalate to exit 2. DRIFT/STALE repo/concern/summary lines are sorted first; bucket lines sorted after. Exit 0=fresh, 1=stale, 2=hard error.
- [`atomic/internal/wiki/stamp.go`](../../../atomic/internal/wiki/stamp.go) — `StampSummary` writes `reflects_rev` from `git rev-parse HEAD`; `StampConcern` writes `reflects:` YAML list (each entry `<id>@<fp>`); `StampKnowledge` writes `sources:` YAML list — caller supplies `"<bucket>/<relpath>@<sha256hex>"` entries verbatim, code writes them. `resolveFingerprint` handles three id types: `knowledge/<topic>.md` → sha256 of `wiki/knowledge/<topic>.md` content; indexed repo → sha256 of `signals.md` content; summarized repo → git HEAD SHA. Unresolvable id silently skipped.
- [`atomic/internal/wiki/`](../../../atomic/internal/wiki) (registry, staleness, linkify) — `RegisterWiki` writes `wiki/index.md` path into `<wikis>` block in `~/.claude/CLAUDE.md` (dedup by `filepath.Abs`+`filepath.Clean`, never touches `<atomic>`); `wiki.CheckStaleness` (injected runner+clock seam, reads `generated` age + `.dirty`, no git); `atomic wiki mark-dirty` (path-prefix check → touch `.dirty`); `LinkifyWiki` rewrites path citations in `wiki/` as file-relative markdown links.
- [`atomic/internal/wiki/wiki_test.go`](../../../atomic/internal/wiki/wiki_test.go) — 20+ tests: happy path, status derivation, summarized preservation/downgrade/disk-discovery (single-file and domain-split), signals-win-over-summary, collision refusal (two cases), stable sort, `## Members` links per status, idempotent re-scan narrative preservation, git-init-skip on re-run.
- [`atomic/internal/wiki/bucket_test.go`](../../../atomic/internal/wiki/bucket_test.go) — unit tests for bucket core: `WalkBucket`, `RegisterBucket`, `BucketDiff`, `bucketDiffReadOnly`, `PromoteBucket`.
- [`atomic/internal/wiki/stale_buckets_test.go`](../../../atomic/internal/wiki/stale_buckets_test.go) — tests for bucket staleness section in `Stale`: `STALE bucket <name>` emission, no-block produces zero bucket lines, `declined="true"` block produces zero bucket lines, walk error escalates to exit 2.
- [`atomic/internal/wiki/stamp_knowledge_test.go`](../../../atomic/internal/wiki/stamp_knowledge_test.go) — tests for `StampKnowledge`: writes `sources:` list, file-absent returns error, non-conforming topic name skipped by CLI.
- [`atomic/internal/wiki/bucket_cli_test.go`](../../../atomic/internal/wiki/bucket_cli_test.go) — integration tests for `wikiBucketAction` dispatch: `add`, `list`, `diff`, `promote` verbs via `wikiAction`.

## Coupling

- **signals domain** — `atomic signals scan --out <dir>` redirect is a direct dependency of wiki-output mode; changes to that flag's behavior or output format propagate to the no-signals summarization path. The `signals-gate` partial (composed by all ship verbs) calls `atomic wiki mark-dirty` after signals refresh — adding a ship verb without signals-gate breaks the drift marker.
- **signals domain** — `atomic-signals-inferrer` is dispatched in wiki-output mode by `/refresh-wiki`; the agent's prompt contract (both `target_repo` + `wiki_dir` required, fail loud on partial args) is a cross-domain dependency. The agent is also dispatched in bucket-synthesis mode per bucket with a non-empty diff; that mode writes pages under `wiki/knowledge/`.
- **config domain** — `wiki.CheckStaleness` reads `<wikis>` from `~/.claude/CLAUDE.md`; `atomic claude install` also writes to that file. Changes to the `<wikis>` block format require coordinated updates in both. `atomic-claude-merger` is instructed to preserve the `<wikis>` block verbatim on merge. `bucket add` writes a `## Capture surfaces` section to the realm [`CLAUDE.md`](../../../CLAUDE.md) — a separate file from `~/.claude/CLAUDE.md`.
- **bundle domain** — [`commands/refresh-wiki.md`](../../../commands/refresh-wiki.md) and [`agents/atomic-signals-inferrer.md`](../../../agents/atomic-signals-inferrer.md) are rendered artifacts; touching their templates requires `make render` + `make bundle`. The wiki Go package is part of the embedded binary.
- **workflow domain** — `/refresh-wiki` invokes `atomic-commit` skill for the commit message offer; wiki commit is offered, never automatic (axiom 3).

## Conventions worth knowing

- `classifyMembers` rule 3 (disk-discovery) makes `summarized` reachable on first derivation. Before the 2026-06-11 amendment, `summarized` was unreachable without a prior index entry — nothing ever wrote the status initially. The fix: after `/refresh-wiki` writes summaries, the closing re-scan calls `discoverSummary` and picks them up.
- `discoverSummary` looks for `wiki/repos/<name>.md` (single-file form) or `wiki/repos/<name>/` containing at least one `.md` (domain-split form). `<name>` is `filepath.Base(rel)`. Domain-split summary path stored as `repos/<name>/` (trailing slash).
- Fingerprints are always written by code (`atomic wiki stamp`), never the model. For knowledge pages: the model supplies which bucket files contributed (`--sources`); code writes every SHA-256 value. For concerns: model declares cited repo ids (`--cites`); code resolves and writes each fingerprint.
- `<wiki-scan>` block carries `root` and `generated` (date from injected clock) on the open tag. Membership and status only — no fingerprints in the block. `generated` doubles as the neglect baseline read by `wiki.CheckStaleness`.
- `<wiki-buckets>` block in `wiki/index.md` carries `<bucket name="…" path="…"/>` entries. `declined="true"` attribute on the open tag means the user declined the bucket-add offer; the block exists but has no entries. `spliceBucketEntry` removes the attribute when a real bucket is added.
- Bucket manifests live at `wiki/.buckets/<name>/` with three files: `current` (debugging artifact, never read back as state), `baseline` (what `diff` compares against), `previous` (prior baseline). `PromoteBucket` always recomputes the walk live — never reads a previously written `current` file.
- `bucketDiffReadOnly` (used by `list` and `stale`) never writes `current`. `BucketDiff` (used by `diff` verb) writes `current`. This is intentional: `list` and `stale` are status verbs with no side effects.
- `atomic wiki stamp`, `atomic wiki mark-dirty`, and `atomic wiki bucket` are internal or restricted helpers. `atomic wiki linkify` and `atomic wiki scan`/`stale` are user-facing and surfaced in `/atomic-help`. `bucket add/list/diff/promote` are user-facing; `stamp --knowledge` is called only by `/refresh-wiki`.
- Session-start staleness nudge threshold defaults to 30 days (memory, axiom 2 — conversational nudge, not config).
- `atomic wiki scan` is idempotent: re-running regenerates only the `<wiki-scan>` block and `## Members` section; content outside those managed zones and all files under `repos/`/`concerns/`/`knowledge/` are untouched.
- Root collision: if `wiki/` exists but `index.md` is absent or lacks a `<wiki-scan>` marker, `scan` refuses with a non-zero exit and names the path in the error.
- Knowledge topic filenames must match `[a-z0-9][a-z0-9-]*\.md` (kebab-case). Non-conforming names are rejected by `wikiStampAction` with exit 0 (skip, not error).
- Concern files may cite knowledge pages as `knowledge/<topic>.md` in `--cites`; `resolveFingerprint` computes sha256 of the knowledge page file content (not a git revision). `StampConcern`'s resolution uses `wikiDir` as root for knowledge-page ids and `realmRoot` for repo ids.
